DBMS : 요리사가 조리


=> 요리사(DBMS)는 주방(데이스베이스)에서 카레를 만들기위해
필요한 재료(데이터)를 꺼낸다.

배우게 될 내용
1. 데이터 저장
2. 데이터 검색
3. 데이터 삭제
4. 데이터 수정
5. 데이터 보안 및 무결성 유지 
: 데이터를 안전하게 보호하고, 데이터를 일관성있게 유지
+ 자격증 문제풀이 


프로그래밍 언어 : C, java, C#, C++ 

SQL : 관계형 데이터베이스에서 "데이터를 다루기 위해 사용하는 언어"

-DDL(Data Definition Language)
 =>  데이터베이스를 정의하는 언어 
      데이터를 생성, 수정, 삭제하는등의 데이터의 전체 골격을 결정하는 역할

create : 데이터베이스 , 테이블등을 생성
alter : 테이블 수정
drop : 데이터 베이스 , 테이블 삭제 

- DML(Data Manipulation Language)
=> 정의된 데이터 베이스에 입력된 레코드를 조회하거나 수정, 삭제하는 등의 역할이다.
select  : 데이터 조회
insert : 데이터 삽입
update :  데이터 수정
delete :  데이터 삭제


- DCL(Data Control Lanaguage)

 =>  데이터 베이스에 접근하거나 객체에 권한을 주는 등의 역할
 grant : 특정 데이터베이스 사용자에게 특정 작업에 대한 수행 권한을 부여
 revoke :  특정 데이터베이스 사용자에게 특정 작업에 대한 박탈, 회수
 commit :  트랜직션의 작업을 저장
 rollback : 트랜직션의 작업을 취소, 원래대로 복구

관계형 데이터 베이스(RDB _ Relational Database)
=> 데이터를 표(table)의 형태로 저장하는 데이터 베이스 종류

=? 관계형 데이터 베이스는 말 그대로 
데이터 모델에 기초를 둔 데이터 베이스이다.

=> 데이터 설계는 모든 데이터를 2차원 테이블 형태로 표현한 뒤 각
테이블 간의 관계를 정의하는 것이다.
