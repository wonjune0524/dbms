Having절

sql에서 group by절과 함께 사용되는 구문이고, 그룹화 된
데이터에 대한 조건을 필터링하는데 사용된다.

select
from
where
group by
having

having절은 논리적으로 group by절 이후에 사용되기 때문에
그룹핑 후에 가능한 집계 함수로 조건을 부여할 수 있다.


where절
적용 시점 : group by가 실행되기 이전
적용 대상 : 개별 행(row)에 대한 조건
주요 사용 : 행 단위에서 특정 조건을 만족하는 데이터를
필터링하는데 사용

having절
적용 시점 : group by가 실행 후 적용
적용 대상 : 그룹화된 결과에 대한 조건
주요 사용 : 집계 함수의 결과를 기준으로 그룹을 필터링
할 때 사용


Join 이란?
MySQL에서 join은 두 개 이상의 테이블을 연결하고,
새로운 결과를 생성한다.
이 때, 연결되는 기준은 일반적으로 두 테이블의 공통된 칼럼임.

Join의 특징
1. 조인하는 테이블에는 같은 값을 가진 칼럼이 필요하다.
2. 두 개 이상의 테이블을 join한다.
3. join할 때, 테이블에 대한 별칭을 사용해주면 좋다.
4. join 칼럼을 비교하는 조건이 필요하다.

--------------------------------------------------

Inner Join(A와 B의 테이블 교집합)
join하는 테이블의 on절과 일치하는 결과만 출력

- select [column_list]
조회하고자 하는 칼럼을 지정한다.
원하는 테이블의 칼럼을 콤마(,)로 구분하여 나열한다.
- from table1
첫 번째 기준이 되는 테이블을 지정한다.
- inner join table2
결합할 두 번째 테이블을 지정한다.
- on table1.column_name = table2.column_name
On 키워드는 두 테이블을 결합할 때 사용할 조건을 정의
이 조건은 일반적으로 두 테이블의 공통된 값을 가진 칼럼을 기준으로 한다.

Outer Join
(left / right / full outer join 총 3가지)

Left outer join : 왼쪽 테이블을 기준으로 모든 행을 포함하고,
오른쪽 테이블의 일치하는 데이터가 있을 경우 그 데이터를 가져온다.
만약, 오른쪽 테이블에 일치하는 데이터가 없다면 그 부분은 null값으로 반환한다.

select [칼럼 리스트]
from 왼쪽 테이블
left outer join 오른쪽 테이블
(값이 없다면 null 포함된다)
on 조건
